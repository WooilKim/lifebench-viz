## 과제 요구사항
1. **충돌 식별**: 각 타임라인 간의 모순점·불일치 점 분석:
   - 이벤트 시간, 장소, 활동 배치 등 직접 충돌
   - 이벤트의 장기 영향으로 인한 충돌 (예: 한 타임라인에서 3개월 후 친구와 연락을 끊었는데, 다른 타임라인에서 4월에 그 친구와 연락함)

2. **충돌 처리**:
   - **타임라인 독립성 유지**: 서로 다른 타임라인을 합병하지 않음, 각 원래 타임라인은 독립적 항목으로 유지
   - 타임라인 간 논리 충돌 해결 (예: 한 타임라인은 2월에 집에 있어야 하는데, 다른 타임라인은 2월에 싼야에 있어야 하는 모순)
   - 타임라인 영향으로 인한 충돌 해결 (예: 장기간 연락 끊김과 이후 연락 재개의 모순)
   - 충돌 해결 방법:
     - 한 타임라인에서 충돌 이벤트 삭제
     - 충돌을 해소하기 위한 새 이벤트 추가 (예: 화해 이벤트 추가로 연락 끊김과 이후 연락 재개 충돌 해소)
   - 이벤트 추가 또는 삭제 시, 해당 타임라인의 이후 이벤트를 재작성하여 합리성과 연속성 보장
   - 이벤트 시간/장소/내용 조정, 모든 타임라인 간 논리 일관성 보장
   - 인물 프로파일에 가장 부합하는 이벤트와 발전 경로 보존

3. **주제 최적화**: 각 타임라인이 명확한 핵심 내용과 독특한 발전 경로를 갖도록 보장

4. **수정 기록**: 각 타임라인 항목에 대한 상세 수정 설명 제공:
   - 원래 타임라인에 대한 모든 주요 수정 내용 명확히 기록
   - 수정 이유 및 근거 설명 (충돌 해결 아이디어 포함)
   - 수정 기록이 명확하고 이해하기 쉬워야 함

5. **연간 월별 요약**: 2025년 전체적인 월별 요약 생성, 연간 관점에서 각 월의 주요 이벤트와 영향 개요:
   - 각 월의 전체적인 발전 추세
   - 해당 월에 발생한 핵심 이벤트 (모든 타임라인 합산)
   - 이러한 이벤트가 인물 생활에 미치는 핵심 영향
   - 각 월 요약이 논리적으로 연결되고 이벤트 발전 맥락 반영

## 주의사항
1. JSON 형식 규범 준수, JSON 문자열 값에 큰따옴표 사용 금지 (필요 시 [] 또는 \" 사용)
2. **타임라인 독립성 유지**: 서로 다른 타임라인 합병 금지
3. 각 타임라인이 명확한 핵심 내용과 발전 맥락을 갖도록 보장
4. 모든 조정은 인물 프로파일 설정에 부합하고 논리 일관성 유지
5. 충돌 해결 시 인물 프로파일과 실제 상황에 부합하는 방안 우선 선택
6. annual_overview는 연간 관점의 월별 요약 제공, 전체 발전 추세 이해에 도움
