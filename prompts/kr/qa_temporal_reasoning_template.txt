시간 순서, 빈도, 시간 간격 등 시간 관련 요소를 결합해야 답할 수 있는 질문을 생성하세요.
질문 유형은 균등 분포: 절반은 주관식, 절반은 객관식.

**시간 추론 질문 정의**: 이벤트 데이터의 시간 속성(발생 시간, 지속 시간, 시간 간격, 빈도 등)을 기반으로 이벤트 간 시간 관계를 분석하여 특정 결론을 추론하는 질문.

질문 설계 원칙:
1. W원칙(시간 차원 집중) 참조. 예:
   - "2025년 8월 처음 헬스장에 간 후 두 번째로 간 것은 얼마 만인가요?"
   - "2025년 9월에 참가한 모든 활동 중 가장 먼저 시작한 것은 무엇인가요?"
2. 이벤트의 시간 속성을 직접 기반으로 질문, 시간 요소가 답변에 핵심 역할을 함.
   예: "2025년 9월 참가 활동 중 가장 먼저 시작한 것은?"
3. 최소 정보 원칙 준수: 답변 가능한 전제에서 최소 정보량 포함, 과도한 검색 힌트 제공 금지.
4. 정보 모호화 처리 기법 적용:
   - 시간 모호화 전략:
     * 이벤트가 여러 시간대에 발생할 수 있다면, 월(예: "2월")로 우선 모호화
     * 같은 달에 동일 이벤트가 여러 번 발생할 수 있다면, 날짜 범위(예: "2월 5~10일")로 모호화
     * 입력 데이터 범위 내 혼동 유발 유사 이벤트가 없으면 원래 날짜 사용 가능
     * 위 방법으로도 정확한 위치 특정이 어려우면 구체적 날짜 명시 (가능한 날짜 범위 사용, 일 단위 특정 최소화)
   - 이벤트 설명 단순화: 이벤트 위치 파악에 필요한 핵심 정보만 보존.
     예: "2025년 8월 31일 아파트 화원에서 산책 중 이웃을 우연히 만남"을 "화원 산책 중 이웃과 우연히 만남"으로 단순화
   - 등가 대체: 더 간결한 표현으로 대체. 예: 호칭을 이름으로 대체 ("딸"을 "소미"로 대체)
5. 질문은 일정 수준의 추론 깊이 필요. 단순 시간 정보 추출이 아닌 여러 이벤트의 시간 관계 분석 필요.

질문 설계 전략:
1. 시간 순서 분석: 이벤트 발생 선후 순서 또는 특정 시간대 활동 배치 질문.
   예: "2025년 10월 참가 활동들을 시간 순서대로 나열하면?"
2. 시간 간격 계산: 두 관련 이벤트 간의 시간 차이 또는 간격 일수 계산.
   예: "도서관 첫 방문과 마지막 방문 사이 간격은 며칠인가요?"
3. 빈도 패턴 인식: 반복 발생 이벤트의 빈도와 시간 규칙 식별.
   예: "2025년 3분기 평균 주당 헬스장 방문 횟수는?"
4. 시간 관계 추론: 이벤트 논리 체인을 기반으로 다음 이벤트 추론.
   예: "디즈니랜드 첫 방문 후 이를 기념하기 위해 무엇을 했나요?"
5. 시간 추세 분석: 시간에 따른 이벤트 변화 추세 또는 패턴 분석.
   예: "2025년 7월과 비교해 8월 업무 회의 빈도는 어떻게 변했나요?"
6. 겹치는 이벤트 분석: 동일 시간 구간 내 발생하는 겹치는 이벤트 식별.
   예: "2025년 9월 15일, 일정 충돌이 있었나요?"

예시 질문:
- 주관식: 2025년 8월 처음 헬스장에 간 후 두 번째로 간 것은 얼마 만인가요?
- 객관식: 2025년 2분기 기술 교육 참가 횟수는?
  A. 1회  B. 2회  C. 3회  D. 4회
- 주관식: 7월과 비교해 8월 독서 빈도는 어떻게 변했나요?
- 객관식: 2025년 9월 가장 먼저 시작한 활동은?
  A. 아침 달리기  B. 회의  C. 공부  D. 쇼핑

요구사항:
1. 질문은 반드시 제공된 이벤트 데이터를 엄격히 기반으로, 시간 차원 분석에 집중
2. 질문은 시간 순서, 빈도, 시간 간격 등 시간 요소의 추론 필요성을 충분히 반영
3. 최소 정보 원칙 준수, 필요한 핵심 정보만 포함, 과도한 힌트성 내용 금지
4. 반드시 정보 모호화 처리 기법 적용: 시간 모호화, 이벤트 설명 단순화, 등가 대체
5. 질문은 일정 수준의 깊이와 복잡성 필요. 여러 이벤트의 시간 관계 분석이 필요하며, 단순한 정보 추출 금지
6. 답변은 구체적이고 명확해야 하며, 개방적이거나 정확성 판단이 어려운 질문 금지
7. 객관식은 4개 선택지 제공, 정답은 1개
8. 객관식 선택지 내용은 최소 정보 원칙 준수, 필요한 핵심 정보만 포함, 추가 추론이나 설명 금지
9. 주관식과 객관식 비율은 균형 유지, 각각 총 질문 수의 절반

각 질문에 대해 제공:
- 구체적 질문 내용
- 구체적이고 명확한 답변
- 채점 기준 설계 (만점 10점), 각 채점 기준은 하나의 명확한 사실 정보에 대응

주의사항:
- 채점 기준은 답변과 강하게 결합, 채점 기준 달성 여부는 답변으로 직접 검증 가능해야 함
- 답변 지원이 없는 채점 기준은 무효
- 채점 기준은 하나일 수 있음 (직접 10점)
- 채점 기준은 답변 텍스트에 대해서만 설계, 이벤트 데이터에 대해 설계하지 않음
- 답변은 간결하고 정확, 개방형 서술 질문 금지
