원래 이벤트 배치와 지도 검색 결과를 바탕으로, 해당 일의 이벤트에 실제 주소와 이동 시간을 배분하여 개인의 이동 궤적이 실제 시간 논리에 부합하도록 보장합니다.

## 이벤트 조정 핵심 요구사항

1. **이벤트 합리성 검증**: 먼저 모든 이벤트에 역사적 사실, 현재 객관적 상황, 인물 신분 및 행동 논리, 시공간 논리와의 충돌이 있는지 판단. 충돌이 있으면 수정. 그 다음 미래를 가리키는 이벤트 설명 (예: "XX와 내일 모임 논의", "모레 여정 계획" 등 아직 발생하지 않은 일정)이 포함되어 있는지 확인. 수정하기 어려우면 제거 고려. 이후 이벤트 설명과 시간을 합리적으로 수정하여 실제 상황에 부합하도록.

2. **이벤트 실행 보장**: **원래 이벤트 배치를 최대한 따름**. 세부 사항만 보충하고, 시간 조정, 실제 주소 배분. 원래 배치 이벤트는 보존하여 실행해야 하며, 극도로 심각한 충돌이 아니면 원래 배치를 삭제하지 말 것. 실행 방식은 유연하게 조정 가능 (예: 오전 항공편을 저녁으로 변경, 오프라인 티켓 구매를 온라인으로 변경 등). 주어진 실행 이벤트와 설명에 엄격히 따를 필요는 없으나, 당일 실행 효과를 달성해야 함 (예: 어떤 이벤트가 다른 도시에 도착하는 효과를 낸다면, 반드시 이를 배치해야 함. 그렇지 않으면 이후 이벤트와 충돌).

3. **당일 이벤트 참조 처리**: 당일 이벤트 참조의 이벤트가 모두 실행되었는지 보장. 이벤트 및 실행 방식/시간을 유연하게 조정 가능하나, 모든 이벤트가 당일 완료되어야 함. 모든 이벤트 조정은 당일 범위 내에서만 허용, 다른 날짜로 조정 불가.

4. **지도 검색 결과 참조 및 처리 규칙**:
   - 지도 검색 결과는 당일 이벤트 원래 배치 데이터를 기반으로 검색 생성됨. 검색 결과가 완전하지 않을 수 있음 (예: 원래 배치에 학교 방문 이벤트가 있으나 검색 결과에 없을 수 있음). 이것이 원래 배치 이벤트를 삭제하라는 의미가 아님. 스스로 해당 이벤트의 이동 시간, 방식, 상세 장소를 결정해야 함.
   - 처리 전략:
     1. 검색 결과의 장소가 불합리한 경우: 장소 수정, 합리적 이동 방식과 시간 직접 분석
     2. 검색 결과의 교통 방식이 불합리한 경우: 교통 방식 수정, 새 교통 방식에 따른 합리적 이동 시간 추론
     3. 검색 결과의 장소와 교통 방식이 모두 불합리한 경우: 동시에 수정, 합리적 이동 시간 직접 추론
     4. 검색 결과에 없는 원래 배치 이벤트: 합리적인 상세 장소, 이동 방식, 시간 직접 결정
   - **주의: 지도 검색 결과는 참조용. 검색 결과가 불완전하거나 불합리하다는 이유로 원래 배치 이벤트 또는 당일 이벤트 참조의 이벤트를 삭제하면 안 됨.**
   - 이동 세부 사항 명확화: 검색 결과와 직접 추론을 결합하여 이동 과정 명확화. 구체적 이동 시간 (분 단위), 이동 방식 (지하철, 버스, 자가용, 도보 등) 포함.
   - 참조 데이터가 없으면 합리적인 장소와 이동 시간 직접 추론.

5. **장소 전환 명확화**: 각 이벤트 간의 장소 전환 논리 명확화. 장소 전환이 시간 순서, 이동 방식과 일치하도록 보장. 공간 이동의 불합리한 상황 방지.

6. **이벤트 배치 합리적 조정, 그러나 핵심은 실제 주소와 이동 시간 및 실행 방식 배분이며, 이벤트 삭제가 아님. 새 이벤트 추가는 허용.**

7. 마지막으로 생성된 전일 이벤트 검토. 이벤트 배치와 설명의 합리성, 현실성 보장. 개인의 하루 생활 배치의 진실하고 합리적인 고품질성을 우선 보장. **동시에 모든 원래 배치 이벤트가 보존되고 실행되었는지 보장.**

8. 하루 이벤트 배치를 풍부하게 하여 내용이 더욱 생동감 있고 현실적으로 만들기.

## 출력 형식 요구사항
전일 이벤트를 출력하되, "시간 구간, 이벤트, 장소, 인물, 설명" 형식으로 제시. 시간 구간은 합리적으로 분할. **장소는 시/구/도로/건물/상세 명칭을 포함한 상세 주소.**
